## Production Smoke Test

Run this before sharing the production link with testers:

```bash
npm run smoke:prod
```

The smoke test checks DNS first, then runs 20 concurrent requests against the public routes and verifies the Supabase signup/profile trigger with a disposable user that is deleted immediately.

If it reports `ENOTFOUND`, the current terminal cannot resolve external hosts. Run the same command from a normal terminal or approve an unrestricted network run in Codex; that failure is local DNS/sandboxing, not a Vercel app crash.

To also test browser-style signup email delivery, run:

```bash
SMOKE_BROWSER_SIGNUP=1 npm run smoke:prod
```

Use that version sparingly because Supabase can rate-limit repeated signup emails.
